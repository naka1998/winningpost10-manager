## 1. 画面一覧とルート定義

TanStack Router によるルート定義。型安全なパラメータを活用する。

| パス | 画面名 | 機能参照 | 優先度 | 説明 |
|------|--------|---------|--------|------|
| `/` | ホーム | F08 | P2 | ダッシュボード。MVP時は `/horses` へリダイレクト |
| `/horses` | 馬一覧 | F01 | P0 | 全馬の検索・フィルタ付きデータテーブル |
| `/horses/import` | 読専txtインポート | F01 | P0 | ステップ形式のインポートウィザード |
| `/horses/$horseId` | 馬詳細 | F01 | P0 | 基本情報・年度別ステータス・配合履歴のタブ表示 |
| `/horses/$horseId/pedigree` | 血統ツリー | F02 | P0 | 3ビュー切替の血統表。4世代デフォルト/5世代切替 |
| `/horses/$horseId/growth` | 成長チャート | F04 | P1 | 能力値推移の折れ線グラフ |
| `/lineages` | 系統マスタ一覧 | F06 | P0 | 親系統・子系統の階層表示 |
| `/plans` | 〆配合プラン一覧 | F03 | P0 | プランのカード一覧 |
| `/plans/$planId` | 〆配合プラン詳細 | F03 | P0 | 4軸ダッシュボード |
| `/plans/$planId/breeding/$year` | 年度配合計画 | F03-C | P0 | 年度別の種付け予定一覧 |
| `/breeding-records` | 配合記録 | F05 | P1 | 配合実績の検索・一覧 |
| `/race-plans/$year` | レース計画 | F11 | P1 | 国×距離帯×グレードのマトリクス表示 |
| `/broodmares` | 繁殖牝馬評価 | F12 | P1 | 個別評価 + 血統バランスの2タブ |
| `/settings` | 設定・バックアップ | F07 | P1 | ゲーム設定、DB入出力、プリセット投入 |

### ルートパラメータの型定義

```typescript
// 型安全なパラメータ
type HorseParams = { horseId: string };  // 実際はID（数値文字列）
type PlanParams = { planId: string };
type YearParams = { year: string };
type PlanYearParams = PlanParams & YearParams;
```

### 検索パラメータ（クエリストリング）

| 画面 | パラメータ | 用途 |
|------|-----------|------|
| `/horses` | `?status=現役&lineage=5&sort=name&q=ジュピター` | フィルタ・ソート・検索の永続化 |
| `/breeding-records` | `?mare_id=10&year=2025` | 特定牝馬・年度での絞り込み |
| `/race-plans/$year` | `?country=日` | 国での絞り込み |

## 2. ナビゲーション構造

### 2.1 サイドバーメニュー

```
🏠 ホーム                    /

📋 馬管理
  ├─ 馬一覧                  /horses
  └─ データ取込              /horses/import

🧬 血統・系統
  └─ 系統マスタ              /lineages

📐 配合計画
  ├─ 〆配合プラン            /plans
  └─ 配合記録                /breeding-records

📅 年度管理
  ├─ レース計画              /race-plans/{currentYear}
```

- サイドバーは左固定、幅 240px

- モバイル幅ではハンバーガーメニューで開閉

### 2.2 パンくずリスト

各ページの上部にパンくずリストを表示する。

```
馬一覧 > シーザジュピター > 血統ツリー
〆配合プラン > プランA > 2025年度配合計画
```

ルートの階層構造から自動生成する。馬名・プラン名は動的に取得。

## 3. レイアウト構成

### 3.1 コンポーネント階層

```
<RootLayout>                    ← DB初期化、グローバルErrorBoundary
  <SidebarLayout>               ← サイドバー + メインコンテンツ
    <Breadcrumbs />             ← パンくずリスト
    <Outlet />                  ← 各ページのレンダリング先
  </SidebarLayout>
</RootLayout>
```

### 3.2 RootLayout

- **責務**: SQLite WASMの初期化完了待ち、グローバルエラーバウンダリ
- 初期化中はローディング画面を表示
- DB初期化失敗時はエラー画面（リトライボタン付き）

```typescript
function RootLayout() {
  const { isReady, error } = useDatabaseInit();

  if (error) return <DatabaseError error={error} />;
  if (!isReady) return <LoadingScreen message="データベースを初期化中..." />;

  return <Outlet />;
}
```

### 3.3 SidebarLayout

- **責務**: サイドバー + メインコンテンツエリアのレイアウト
- サイドバーの開閉状態は `useUIStore` で管理

```
┌──────────┬──────────────────────────────────────┐
│          │  パンくずリスト                         │
│  サイド   │──────────────────────────────────────│
│  バー     │                                      │
│          │  メインコンテンツ                       │
│  240px   │  （各ページの <Outlet />）             │
│          │                                      │
│          │                                      │
│──────────│                                      │
│ 年度:2025│                                      │
└──────────┴──────────────────────────────────────┘
```

## 4. 主要コンポーネント設計

### 4.1 PedigreeTree — 血統ツリー（F02）

馬の4〜5世代血統表をCSS Gridで描画する。

**配置先:** `/horses/$horseId/pedigree`

**Props:**
```typescript
interface PedigreeTreeProps {
  horseId: number;           // 起点となる馬のID
  depth?: 4 | 5;            // 表示世代数（デフォルト: 4）
  viewMode?: 'name' | 'lineage' | 'factor';  // 表示モード
}
```

**機能:**
- **CSS Grid レイアウト**: 4世代 = 5列 × 16行（起点1 + 父母2 + 祖父母4 + 曽祖父母8 + 4代前16）。5世代はさらに1列追加
- **3ビュー切替**: ツリー上部のトグルボタンで馬名/系統/因子ビューを切替
  - **馬名ビュー**: 馬名、国旗アイコン
  - **系統ビュー**: 所属子系統名、親系統名
  - **因子ビュー**: 系統タイプ（SP/ST）、保有因子リスト
- **インブリード検出**: 同一祖先を色でハイライト + 「3×4」等のクロス表記をバッジ表示
- **親系統カラーリング**: 親系統ごとに背景色を割当。血脈の偏りを視覚化
- **祖先馬クリック**: クリックでその馬の詳細画面へ遷移

**CSS Grid構造（4世代の場合）:**
```
       1代前    2代前    3代前    4代前
      ┌──────┬──────┬──────┬──────┐
      │      │      │      │ 父父父父│

      │      │      │      │ 父父父母│

      │      │      │      │ 父母父父│
      │      │      │ 父母父├──────┤
      │      │ 父母  │      │ 父母父母│
```

### 4.2 GanttTimeline — 年度タイムライン（F03-B④）

〆配合プランの全タスクを横軸=年度で表示するガントチャート。

**配置先:** `/plans/$planId` 内の4軸ダッシュボード

**Props:**
```typescript
interface GanttTimelineProps {
  planId: number;
  currentYear: number;       // 現在のゲーム内年度（縦線表示）
}
```

**機能:**
- **Rechartsベース**: `BarChart` を横向きに使用。各レーンをグループ化
- **3レーン構成**:
  - 系統確立: 各系統の「確立目標期間」をバーで表示
  - 世代生産: 各世代の馬を「いつ生産するか」のバー
  - 繁殖牝馬: 「現役→繁殖入り」のタイミングバー
- **現在年度マーカー**: 縦の破線で「今ここ」を表示（`ReferenceLine`）
- **色分け**: 完了=緑、進行中=青、遅延=赤、未着手=グレー
- **横スクロール**: タイムライン全体がビューポートを超える場合はスクロール対応

### 4.3 HorseTable — 馬一覧テーブル（F01）

全馬のデータテーブル。フィルタ・ソート・ページネーション付き。

**配置先:** `/horses`

**Props:**
```typescript
interface HorseTableProps {
  initialFilters?: HorseFilters; // URLパラメータからの初期フィルタ
}
```

**機能:**
- **カラム**: 馬名 / 性別 / 生年 / 国 / ステータス / 所属系統 / SP / 戦績
- **フィルタバー**: ステータス（ドロップダウン）、系統（ドロップダウン）、テキスト検索
- **ソート**: 各カラムヘッダーでの昇順/降順切替
- **行クリック**: 馬詳細画面へ遷移
- **表示切替**: 所有馬のみ / 全馬（祖先含む）のトグル

### 4.4 StepProgress — 世代進行ステップバー（F03-B②）

〆配合の世代進行を4〜5ステップのプログレスバーで表示。

**配置先:** `/plans/$planId` 内の世代進行マップ

**Props:**
```typescript
interface StepProgressProps {
  lineType: 'sire' | 'dam';   // 〆父ライン / 〆母ライン
  steps: Array<{
    generation: number;
    horseName: string | null;  // null = 未定
    status: 'completed' | 'in_progress' | 'planned';
  }>;
}
```

**機能:**
- 4ステップ（世代）を横並びで表示
- 完了ステップは塗りつぶし+チェックマーク
- 進行中ステップはアニメーション付きハイライト
- 未着手ステップはグレーアウト
- 各ステップに馬名（or 「未定」）を表示
- 残り世代数をサマリーテキストで表示

### 4.5 ImportWizard — インポートウィザード（F01）

読専txtファイルのインポートをステップ形式で実行。

**配置先:** `/horses/import`

**ステップ構成:**

```
[1. ファイル選択] → [2. 設定] → [3. プレビュー] → [4. 実行結果]
```

1. **ファイル選択**: ドラッグ&ドロップまたはファイル選択ダイアログ。ファイル名・サイズのバリデーション
2. **設定**: インポート年度の指定（D2のスナップショット年度）。通常は読専データ取得時のゲーム内年度
3. **プレビュー**: パース結果の差分表示
   - 新規登録件数（緑バッジ）
   - 更新件数（黄バッジ）: 既存馬の能力値変更
   - 変更なし件数（グレーバッジ）
   - エラー行（赤バッジ）: パース失敗した行の詳細
   - テーブル形式で全行をプレビュー、差分がある行をハイライト
4. **実行結果**: インポート完了サマリー。import_logsへの記録結果

**Props:**
```typescript
interface ImportWizardProps {
  onComplete: (result: ImportResult) => void;
}
```

### 4.6 BreedingEvalCard — 配合評価カード（F05）

配合の評価結果をカード形式で表示。

**配置先:** `/breeding-records`, `/plans/$planId`

**Props:**
```typescript
interface BreedingEvalCardProps {
  evaluation: string;          // A/B/C等
  theories: Array<{
    name: string;              // 理論名
    points: number;            // 爆発力ポイント
  }>;
  totalPower: number;          // 合計爆発力
}
```

**機能:**
- 配合評価ランクを大きく表示
- 成立理論をリスト表示（理論名 + ポイント）
- 合計爆発力をハイライト表示

### 4.7 RacePlanMatrix — レース計画マトリクス（F11）

国×距離帯×グレードの3次元マトリクスで路線被りを可視化。

**配置先:** `/race-plans/$year`

**Props:**
```typescript
interface RacePlanMatrixProps {
  year: number;
}
```

**機能:**
- **行**: 距離帯（短距離/マイル/中距離/中長距離/長距離）
- **列**: 国（日/米/欧）× グレード（G1/G2/G3/OP）
- **セル**: 割り当てられた馬名を表示。馬の適性に合致するセルを薄くハイライト
- **被り警告**: 同一セルに2頭以上の馬が配置されている場合、赤枠+警告アイコン
- **ドラッグ&ドロップ**: 馬をセル間で移動して路線を変更可能（P2拡張）

## 5. レスポンシブ対応方針

デスクトップファーストで設計する。モバイル対応は必須ではないが、最低限のレイアウト崩れ防止は行う。

| ブレークポイント | 対応 |
|----------------|------|
| `>= 1024px`（デスクトップ） | フルレイアウト。サイドバー常時表示 |
| `768px - 1023px`（タブレット） | サイドバーをオーバーレイに変更 |
| `< 768px`（モバイル） | サイドバー非表示。ハンバーガーメニューで開閉。テーブルは横スクロール |

- 血統ツリーは横スクロール対応（4世代で最低800px幅を確保）
- データテーブルは固定列（馬名）+ スクロール列で対応
- Tailwind CSSの `md:` / `lg:` プレフィックスで切替
