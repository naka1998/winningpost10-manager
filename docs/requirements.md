# WP ブリーディングマネージャー 要求整理・要件定義書

> **ドキュメントバージョン:** v0.1 ドラフト
> **最終更新:** 2026-02-28
> **ステータス:** レビュー待ち

---

## 1. プロジェクト概要

### 1.1 目的

Winning Post 10 2026（未発売）のプレイデータを管理・分析し、**〆配合の計画から実行までを一気通貫で支援する**個人用Webアプリケーションを開発する。開発・テスト段階ではWP10 2025のデータを使用する。

### 1.2 背景と課題

過去にWP9でスプレッドシート（Excel）管理を試みたが、以下の限界を経験した。WP10 2026では最初から専用アプリで管理したい。

| 過去の運用（WP9）                         | 具体的なツラさ                                                               |
| ----------------------------------------- | ---------------------------------------------------------------------------- |
| 〆配合の血統ツリーを手動で表に配置        | 4代前までのツリー構造をセル結合で表現しており、組み替えや比較が困難          |
| 年度別シートに各馬のステータスを転記      | ゲーム画面を見ながら手入力。同じ馬が複数年度に散在し、成長の変化を追いにくい |
| 配合評価・爆発力を年度×牝馬の横持ちで管理 | 巨大テーブル。追加・集計が煩雑                                               |
| 系統確立の進捗を備考欄にメモ              | 〆配合のゴールまでに「何年にどの系統を確立すべきか」の全体像が見えない       |

### 1.3 対象ユーザー

- **自分専用**（シングルユーザー）
- 認証は簡易で可（ローカル起動 or 簡易パスワード）

### 1.4 プラットフォーム

- **Webアプリ**（ブラウザで動作）
- レスポンシブ対応は必須ではないが、あると望ましい

### 1.5 ゲームプレイの年間ライフサイクル

アプリの利用タイミングはゲーム内の年間サイクルに沿う。

| 時期     | ゲーム内イベント     | アプリでの作業                                                                                   |
| -------- | -------------------- | ------------------------------------------------------------------------------------------------ |
| **1月**  | 新年度開始           | 読専txtインポート（年度データ更新）。2歳新馬をまとめて登録、3歳以上の現役馬データを更新          |
| **1月**  | レース計画           | 現役馬のレース路線を計画。国・距離・グレードの被りがないようにチェック（F11）                    |
| **5月**  | 種付け時期           | 配合記録（F05）。繁殖牝馬の評価（F12）で生産性・血統バランスを確認。売却候補等のメモを備考に記入 |
| **年末** | 幼駒の取捨・引退判断 | 現役馬→引退/種牡馬/繁殖牝馬のステータス変更。幼駒の残す・売るの判断メモ                          |
| **年末** | 1年の振り返り        | （アプリ操作なし）                                                                               |

> **設計への影響:** 1月のtxtインポート→レース計画が最もヘビーな利用タイミング。この流れがスムーズであることが重要。

### 1.6 将来拡張

- **マルチプロファイル:** 複数セーブデータの並行管理
- **マルチバージョン:** WP10 2027等、将来のバージョンへの対応
- 上記に備え、ゲームバージョン固有のデータ構造（能力値項目、読専txtフォーマット）を差し替えられる設計とする

## 2. 機能要件

### 優先度の定義

| ランク | 意味                                                 |
| ------ | ---------------------------------------------------- |
| **P0** | MVPに必須。WP10 2026の発売までに動いている必要がある |
| **P1** | MVP直後に実装。日常のプレイサイクルで必要になる      |
| **P2** | あると嬉しい。余裕があれば実装                       |
| **P3** | 将来構想。設計段階で拡張余地を残しておく             |

### F01: 馬マスタ管理 [P0]

**目的:** すべての馬（競走馬・繁殖牝馬・種牡馬）を一元登録し、アプリの基盤データとする。

馬に関するデータは性質の異なる3つの領域に分離して管理する。

---

#### D1: 基本情報（馬の「身元」）

馬そのものを識別する不変〜低頻度変更のデータ。1頭につき1レコード。

| 項目         | 型       | 説明                              | 例               |
| ------------ | -------- | --------------------------------- | ---------------- |
| 馬名         | テキスト | 冠名含む                          | シーザジュピター |
| 性別         | 区分     | 牡/牝/セン                        | 牝               |
| 生年         | 数値     | ゲーム内の生まれ年                | 2010             |
| 国（所属）   | 区分     | 日/米/欧                          | 日               |
| 史実馬フラグ | 区分     | 史実馬/自家生産馬                 | 自家生産馬       |
| 牝系名       | テキスト | 母系ライン名                      | プリティポリー系 |
| ステータス   | 区分     | 現役/繁殖牝馬/種牡馬/引退・売却済 | 現役             |

> **設計意図:** ここに入る情報はゲーム進行中にほぼ変わらない。馬の一意な「カード」として機能する。ステータス区分のみ、引退・繁殖入り時に更新される。

---

#### D2: レースステータス（馬の「今」）

年度ごとに変化する能力値・レース関連データ。1頭×年度ごとにスナップショットとして記録する。

**適性・距離**

| 項目     | 型       | 説明               | 例          |
| -------- | -------- | ------------------ | ----------- |
| 芝       | 区分     | 芝適性 ◎/○/×       | ◎           |
| ダ       | 区分     | ダート適性 ◎/○/×   | ○           |
| 芝質     | テキスト | 軽〜重のレンジ表記 | 1-2(1-4)    |
| 距離適性 | テキスト | 下限〜上限         | 1100〜2000m |

**能力値（7項目）**

| 項目 | 型             | 説明           | 例      |
| ---- | -------------- | -------------- | ------- |
| SP   | ランク or 数値 | スピード       | S+ / 79 |
| 力   | ランク or 数値 | パワー         | S+      |
| 瞬   | ランク or 数値 | 瞬発力         | B       |
| 勝   | ランク or 数値 | 勝負根性       | ??      |
| 精   | ランク or 数値 | 精神力         | A+      |
| 賢   | ランク or 数値 | 賢さ           | D       |
| サ   | 数値           | サブパラ総合値 | 69      |

**成長・戦術**

| 項目 | 型           | 説明                   | 例                             |
| ---- | ------------ | ---------------------- | ------------------------------ |
| 成型 | テキスト     | 成長型                 | 早熟                           |
| 脚質 | 複数区分     | 大逃/逃/先/差/追の適性 | 追                             |
| 特性 | 複数テキスト | 保有特性               | 大舞台, 鉄砲, 牝馬混合, 直一気 |

**運用情報**

| 項目     | 型              | 説明                   | 例                 |
| -------- | --------------- | ---------------------- | ------------------ |
| 父馬     | テキスト / 参照 | 種牡馬名               | エルコンドルパサー |
| 主戦騎手 | テキスト        |                        | 田中勝春           |
| グレード | テキスト        | 出走クラス・目標レース | G1 古馬ダート      |
| 戦績     | テキスト        | 出走-勝-2着-3着-着外   | 17戦14勝           |
| 備考     | テキスト        | 自由メモ               |                    |

> **設計意図:** 同じ馬でも年齢を重ねると能力値・適性・脚質が変わる。年度ごとのスナップショットとして保持し、成長曲線の描画やピーク判定に使う。ランク表記と数値の両方を格納できる柔軟な型とする。**データの大半は読専txtインポートで自動投入される。**

---

#### D3: 血統（馬の「血」）

配合理論の判定に使う血統構造データ。**所有馬だけでなく、血統表に登場するすべての祖先馬もデータとして保持する必要がある。**

##### D3-a: 血縁関係（全馬共通）

| 項目       | 型     | 説明               | 例                   |
| ---------- | ------ | ------------------ | -------------------- |
| 父（sire） | 馬参照 | 父馬への自己参照FK | → ケープクロス       |
| 母（dam）  | 馬参照 | 母馬への自己参照FK | → カンパリジュピター |

##### D3-b: 系統情報（全馬共通）

| 項目     | 型             | 説明                 | 例                 |
| -------- | -------------- | -------------------- | ------------------ |
| 所属系統 | 系統マスタ参照 | この馬が属する子系統 | グリーンデザート系 |

> 親系統（例: ダンジグ系）やSP/ST区分は系統マスタ（F06）から引く。

##### D3-c: 因子（種牡馬・祖先馬）

| 項目 | 型       | 説明           | 例               |
| ---- | -------- | -------------- | ---------------- |
| 因子 | 複数区分 | 保有因子リスト | スピード, パワー |

> 画像3に見えるように、4代前の祖先にもそれぞれ因子がある（スピード/パワー/瞬発力/柔軟性/勝負根性/早熟 等）。活力源化配合の判定にはこれらすべてが必要。

##### 所有馬 vs 祖先馬のレコード粒度

血統表に登場する馬は全員D3のデータが必要だが、**D1（基本情報）とD2（レースステータス）は所有馬のみ**で十分。

```
所有馬（自家生産馬・史実馬で所有中）
  → D1（基本情報） + D2（年度スナップショット） + D3（血統）

祖先馬・種牡馬・外部馬（血統表に出るが所有しない）
  → D3（血統）のみ = 馬名 + 父母参照 + 系統 + 因子
```

> **設計意図:** 血統ツリーの描画深度は、登録されている祖先データの範囲で表示する。所有馬とその父馬程度のデータがあれば基本的な運用は可能。より深い血統ツリー（4〜5代）が必要な場合は、必要に応じて祖先馬を手動追加する。**読専txtから父馬名・父系（系統名）・母馬名・牝系が自動取得できるため、1代分の父母リンクと系統情報はtxtインポートで自動構築される。**

#### D1 / D2 / D3 の関係

```
┌─────────────────────────────────────────────────────┐
│  馬テーブル（全馬共通）                                 │
│  馬名 + D3: 父母参照・系統・因子                        │
│  → 所有馬も祖先馬も全員ここに入る                        │
├─────────────────────────────────────────────────────┤
│  所有馬のみ↓                                          │
├─────────────────────────────────────────────────────┤
│  D1: 基本情報（馬の身元）                              │
│  性別・生年・国・牝系・ステータス                         │
│  → 所有馬 or 詳細を把握している馬のみ                    │
├─────────────────────────────────────────────────────┤
│  D2: レースステータス（馬の今）                          │
│  能力値7項目・適性・脚質・特性・成型・戦績                │
│  → 所有馬 × 年度ごとにスナップショット                   │
└─────────────────────────────────────────────────────┘

例: シーザジュピターの血統表には約30頭が登場するが、
    D1/D2が必要なのはシーザジュピター本人だけ。
    残りの祖先馬はD3（馬名・父母・系統・因子）のみ。
```

#### 機能詳細

- 馬の新規登録・編集・削除
- 馬名・系統・適性等での検索・フィルタ
- **読専txt取り込み（メインの登録手段）:** メモリエディタ「読専」から出力されるタブ区切りtxtファイルを取り込み、D1・D2・D3を一括登録・更新
- 馬の詳細画面から「年度別ステータス」「血統ツリー」「配合履歴」へのナビゲーション

#### 読専txtインポート仕様

年度ごとに読専から出力したtxtを取り込むのが**基本のデータ登録フロー**。手入力はtxtでカバーできない部分のみ。

**txtから取り込む項目とデータ領域のマッピング:**

| txt列名  | マッピング先        | データ領域       |
| -------- | ------------------- | ---------------- |
| 馬名     | 馬名                | D1               |
| 国       | 国（所属）          | D1               |
| 年       | 馬齢                | D1（生年を逆算） |
| 性       | 性別                | D1               |
| 芝 / ダ  | 芝適性 / ダート適性 | D2               |
| 芝質     | 芝質                | D2               |
| 距離適性 | 距離適性            | D2               |
| SP       | SP（スピード）      | D2               |
| 力       | パワー              | D2               |
| 瞬       | 瞬発力              | D2               |
| 勝       | 勝負根性            | D2               |
| 精       | 精神力              | D2               |
| 賢       | 賢さ                | D2               |
| サ       | サブパラ総合値      | D2               |
| 成型     | 成長型              | D2               |
| 脚質     | 脚質                | D2               |
| 特性     | 特性                | D2               |
| 戦績     | 戦績                | D2               |
| 父馬     | 父馬名              | D3               |
| 父系     | 父の所属系統        | D3               |
| 母馬     | 母馬名              | D3               |
| 牝系     | 牝系名              | D1 / D3          |

**txtでカバーできず手入力が必要な項目:**

- 祖先馬の血統情報（父母の父母…4代以上遡る部分）
- 因子（txtに含まれない）
- 系統確立イベント
- 繁殖牝馬としてのステータス変更

**取り込み時の処理:**

- 馬名で既存レコードと照合し、存在すれば更新、なければ新規作成
- 取り込み時にインポート年度を指定 → D2のスナップショット年度として使用
- 能力値の「S+(0)」等の表記からランク部分「S+」を抽出（括弧内の数値は変動値として別途保持可能）

### F02: 血統ツリー・家系図ビジュアライザー [P0]

**目的:** 任意の馬を起点に、4〜5代前までの血統をツリー形式で可視化する。

> 📌 **ユーザー最優先機能。** 現状の〆配合シートで最もツラい作業を解消する。

#### 機能詳細

- 指定した馬を起点に、父方・母方を再帰的に辿り**4代（オプションで5代）の血統表**を描画
- 各祖先ノードに表示する情報:
  - 馬名
  - 所属系統（親系統 / 子系統）
  - 保有因子
  - 系統確立フラグ（確立済 or 未確立）
- **インブリード検出:** 同一祖先が複数箇所に出現する場合、ハイライト表示し「3×4」等のクロス表記を自動生成
- **系統の色分け:** 親系統ごとに色を割り当て、血脈の偏りを視覚的に把握
- ツリーからの祖先馬クリックで、その馬の詳細画面へ遷移

#### 表示モード（ゲーム画面の3ビューを再現）

| モード         | 各ノードに表示する情報                                | 用途                                 |
| -------------- | ----------------------------------------------------- | ------------------------------------ |
| **馬名ビュー** | 馬名、国旗                                            | 血統構成の確認、祖先の特定           |
| **系統ビュー** | 所属系統名（子系統）、親系統                          | 血脈活性化・系統の偏りチェック       |
| **因子ビュー** | 系統タイプ（SP系/ST系）、系統規模（零細等）、保有因子 | 活力源化・SP昇華等の配合理論チェック |

> いずれのモードでも**インブリード検出**（同一祖先のハイライト＋クロス表記）と**親系統ごとの色分け**は共通で適用する。

### F04: 年度別ステータス & 成長トラッキング [P1]

**目的:** 各馬の能力値を年度ごとに記録し、成長曲線を可視化する。

> 📌 **ユーザー第3優先。** 「この馬はいつピークを迎えるか」を判断する材料。

#### データ

F01のD2（レースステータス）がそのまま本機能のデータ源。年度×馬ごとのスナップショットを蓄積する。

#### 機能詳細

- 年度ごとのステータス入力フォーム（ゲーム画面を見ながらの手入力に最適化）
- **成長チャート:** 特定の馬を選択すると、SP・パワー等の推移を折れ線グラフで表示
- **年度一覧ビュー:** 特定年度の全所有馬を一覧表示（現在の年度別シートに相当）
- 能力値はランク表記（S+/A/B+等）と数値の両方に対応
  - 読専txtでは「S+(0)」のように両方が取得可能
  - ランク⇔数値の変換テーブルをゲームバージョン設定として保持

### F05: 配合記録 [P1]

**目的:** 種牡馬×繁殖牝馬の配合結果（成立理論・爆発力）をゲームで確認した値で記録し、蓄積する。

> 配合理論の自動計算はしない。ゲーム側で確認した結果をアプリに転記して記録する方針。

#### 機能詳細

- 種牡馬と繁殖牝馬を選択し、配合結果を記録:
  - 配合評価（A/B/C等）
  - 成立した理論名と個別爆発力（手入力）
  - 合計爆発力
  - 備考（気づいたこと等）
- 配合履歴の検索・一覧表示
- 特定の繁殖牝馬の配合履歴を時系列で表示（「この牝馬は過去にどんな種牡馬と付けて、結果はどうだったか」）

---

### F11: レース計画 [P1]

**目的:** 年始に現役馬のレース路線を計画し、**国・距離・グレードの被りを可視化**して最適な分散配置を支援する。

> 📌 ゲーム内1月の主要作業。自分の馬同士で路線が被るともったいないので、一覧で比較して調整する。

#### 機能詳細

- 現役馬一覧を国・距離帯・グレード（G1/G2/G3/OP等）でマトリクス表示
- 各馬に「目標路線」を割り当て（例: 日本・中距離・G1）
- **被りチェック:** 同じ国×距離帯×グレードに複数馬が配置されている場合、警告表示
- 馬の適性（芝/ダ・距離適性）から、走れる路線の候補をハイライト
- 路線割り当ての保存・年度ごとの履歴管理

### F12: 繁殖牝馬評価 [P1]

**目的:** 繁殖牝馬の生産性を「個別」と「全体」の2軸で評価し、売却判断や血統バランスの偏り防止を支援する。

> 📌 5月の種付けシーズンや年末の取捨判断で活用。「この牝馬はもう売っていいか」「同じ系統の牝馬が多すぎないか」を判断する材料。

#### 12-A: 馬ごとの評価（個別）

繁殖牝馬1頭ごとの生産実績を一覧で表示。

| 表示項目     | 説明                       |
| ------------ | -------------------------- |
| 馬名         | 繁殖牝馬名                 |
| 年齢         | 現在の年齢                 |
| 繁殖入り年度 | いつから繁殖に上がったか   |
| 産駒数       | これまでに生んだ頭数       |
| 現役産駒数   | うち現役で走っている頭数   |
| 主な産駒成績 | 産駒のG1勝利数等のサマリー |
| 備考         | 売却候補メモ等             |

> ソート・フィルタ可能。「繁殖入りから5年で産駒0頭」のような非効率な牝馬を素早く発見できる。

#### 12-B: 全体評価（血統バランス）

繁殖牝馬全体の血統構成を可視化し、特定系統への偏りを防ぐ。

| 表示項目         | 説明                                                              |
| ---------------- | ----------------------------------------------------------------- |
| 系統別の牝馬頭数 | 円グラフまたは棒グラフ。「ミスタープロスペクター系の牝馬が40%」等 |
| 牝系別の牝馬頭数 | 同上。母系ラインの偏りチェック                                    |
| 父馬別の牝馬頭数 | 同じ種牡馬の娘が多すぎないかチェック                              |

> 「この系統の牝馬がもう1頭必要」「この系統は余っているから売れる」の判断に使う。

### F06: 系統マスタ管理 [P0]

**目的:** 親系統・子系統のマスタデータを管理する。血統ツリー（F02）の表示の基盤。

> 系統確立の追跡やニックスはゲーム内で確認すれば十分なため管理しない。

#### 管理する属性

- 系統名
- 系統区分（親系統 / 子系統）
- 親系統への参照（子系統の場合）
- 系統タイプ（SP系 / ST系 / 無印）

#### 機能詳細

- 系統の登録・編集
- 系統の親子関係の変動記録（確立で子系統が独立した場合等）
- 初期データとしてWP10のデフォルト系統をプリセット提供
- 系統名での検索

### F07: データ入出力 [P1]

**目的:** 読専txt以外のインポート/エクスポート手段と、バックアップ機能を提供する。

#### 機能詳細

- **CSVエクスポート:** 任意のビュー（馬一覧、年度別ステータス等）をCSV出力
- **データバックアップ / リストア:** DB全体のダンプ・復元
- **プリセットデータ投入:** WP10の系統マスタ・主要祖先馬のD3データを初期投入

### F09: パフォーマンス分析 [P3]

### F10: マルチプロファイル [P3]

- プロファイル間で馬マスタの共通参照は行わない（バージョン間でパラメータが異なるため）

## 3. 非機能要件

### 3.1 パフォーマンス

- 血統ツリーの描画: 登録済み祖先の範囲で即座に表示
- 馬の検索・フィルタ: 500頭規模で体感遅延なし
- 読専txtインポート: 数百行を数秒以内で処理

### 3.2 データ規模の想定

| エンティティ               | 想定件数（1プロファイル）   | 備考                            |
| -------------------------- | --------------------------- | ------------------------------- |
| 馬（全体）                 | 200 〜 500頭                | 所有馬＋その父母（1代分）の累積 |
| うち所有馬（D1/D2あり）    | 20頭/年度、累積で100〜300頭 | 現役＋引退・繁殖入り含む        |
| うち父母・祖先馬（D3のみ） | 50 〜 200頭                 | 所有馬の父馬＋プリセット分      |
| 年度別ステータス（D2）     | 500 〜 2,000件              | 所有馬×年度（20頭×数十年度）    |
| 系統マスタ                 | 100 〜 200系統              | 親系統＋子系統                  |
| 配合記録                   | 100 〜 500件                |                                 |

### 3.3 データ永続性

- ブラウザを閉じてもデータが消えないこと
- 手動バックアップ機能（F07）でデータ損失リスクを軽減

### 3.4 拡張性

- 配合理論の記録項目をゲームバージョンごとに追加・変更可能な設計
- 能力値のランク体系（S+/A/B+等）をバージョンごとに定義可能
- 読専txtのフォーマット変更に対応できるパーサー設計

## 4. 技術方針（案）

> 最終的な技術選定は設計フェーズで確定するが、方向性として。

### 4.1 アーキテクチャ候補

| 方式                      | 概要                             | メリット             | デメリット             |
| ------------------------- | -------------------------------- | -------------------- | ---------------------- |
| **A: フルスタックSPA**    | React/Vue + Node.js + PostgreSQL | 柔軟性が高い、本格的 | 構築・運用コストが高い |
| **B: ローカルファースト** | React + SQLite (WASM)            | サーバー不要、即起動 | DB規模に制限あり       |
| **C: Electron/Tauri**     | デスクトップアプリとして配布     | オフライン完結       | Web配信できない        |

個人用途かつデータ規模を考慮すると、**方式B（ローカルファースト）** が有力。SQLite WASMでブラウザ内にDBを持ち、IndexedDBに永続化するアプローチ。サーバー運用が不要で、データは完全にローカルに留まる。

### 4.2 血統データの設計方針

- 馬テーブルに `sire_id` / `dam_id` の自己参照外部キーを持つ
- 4〜5代の血統展開はWITH RECURSIVE (CTE) で実現可能
- インブリード検出は展開した血統表上での重複チェック
- 系統確立の状態は年度ごとのスナップショットとして保持

### 4.3 配合データの方針

- 配合理論の自動判定は行わない（ゲーム側で確認した結果を記録する方針）
- 成立理論名・爆発力は自由入力フィールドとし、ゲームバージョンに依存しない柔軟な構造とする
- 将来的に自動判定を追加する場合に備え、血統データ（D3）は判定に必要な情報を保持しておく

## 付録A: 用語集

| 用語                   | 説明                                                              |
| ---------------------- | ----------------------------------------------------------------- |
| 〆配合（しめはいごう） | 最終的に目指す理想の配合。複数世代にわたる血統設計のゴール        |
| 爆発力                 | 配合時に成立する理論の合計ポイント。高いほど優秀な産駒が出やすい  |
| 配合評価               | 配合の総合評価ランク（1〜5段階等）                                |
| 系統確立               | ゲーム内で特定の種牡馬系統が独立した系統として認定されるイベント  |
| 親系統                 | 大きな分類の系統（例: ナスルーラ系）                              |
| 子系統                 | 親系統から派生した小分類（例: ナスルーラ系のグレイソヴリン系）    |
| SP系 / ST系            | スピード系 / スタミナ系。系統のタイプ分類                         |
| インブリード           | 血統表内で同一の祖先が複数回出現すること（近親配合）              |
| アウトブリード         | 5代以内にインブリードが存在しない配合                             |
| 血脈活性化配合         | 4代前の8系統がすべて異なる場合に成立                              |
| 因子                   | 各馬が保有する能力因子（スピード/スタミナ/瞬発/根性/パワー/柔軟） |
| ニックス               | 特定の系統同士の相性ボーナス                                      |
| 牝系                   | 母方の血統ライン（例: プリティポリー系）                          |
| 子出し                 | 繁殖牝馬の産駒能力への影響度パラメータ                            |
